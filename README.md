Проект: Эмулятор системы приточной вентиляции (ThingsBoard)
Описание
Данный проект представляет собой эмулятор приточно-вытяжной установки, интегрированный с платформой ThingsBoard. Система в реальном времени передает телеметрию и обрабатывает аварийные ситуации.

Выполнение требований задания
Текущие параметры: На дашборде реализовано отображение температуры (приток/улица), оборотов вентилятора (RPM) и состояния фильтра через цифровые и аналоговые виджеты.

Графики: Настроены графики для мониторинга динамики температур и давления (засорения фильтра).

Статусы и аварии: Реализована индикация статуса работы (RUN) и автоматическая генерация алармов при достижении критического уровня засорения фильтра (>90%) через цепочки правил (Rule Chain).

Эмуляция данных: Скрипт на Python имитирует работу оборудования, включая шум датчиков и вероятностные инциденты.

Технологический стек
ThingsBoard: Платформа управления IoT (развернута в Docker).

Python: Реализация логики эмулятора.

MQTT: Протокол обмена данными.

Docker / Docker Compose: Оркестрация.

Что реализовано (отчет для HR)
Инфраструктура: Развернут локальный сервер ThingsBoard через docker-compose.

Эмулятор: Создан Python-скрипт (emulator.py), который отправляет телеметрию по протоколу MQTT и имитирует нештатные ситуации.

Логика: Настроены Rule Chain для автоматической обработки аварий. Система реагирует на превышение порога filter_clog и фиксирует аварию в виджете.

Сложности: Основной задачей было создание связи между входящей телеметрией и движком алармов ThingsBoard (Rule Chain). Решено путем фильтрации данных через Script Node с последующей передачей в Create Alarm Node.

Инструкция по запуску
Разверните ThingsBoard: docker-compose up -d.

Создайте устройство в ThingsBoard, скопируйте Access Token.

Установите библиотеку: pip install paho-mqtt.

Запустите эмулятор: python3 emulator.py (не забудьте указать свой TOKEN)
