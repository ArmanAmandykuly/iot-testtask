# 🛠️ IoT-Эмулятор приточной вентиляции

Эмулятор системы приточно-вытяжной установки, интегрированный с платформой **ThingsBoard**. Система в реальном времени транслирует телеметрию и обрабатывает нештатные ситуации.

---

## 📊 Основные возможности
* **Real-time телеметрия**: Отображение температур, оборотов вентилятора (RPM) и состояния фильтра.
* **Аналитика**: Графики динамики температур и перепада давления (засорения фильтра).
* **Мониторинг аварий**: Автоматическая генерация алармов при достижении критических значений (настроено через `Rule Chain`).
* **Интеллектуальная эмуляция**: Python-скрипт с имитацией шума датчиков, вероятностными сбоями и циклом обслуживания.

## ⚙️ Технологический стек
* **ThingsBoard**: IoT-платформа (развертывание через Docker).
* **Python**: Реализация логики эмуляции.
* **MQTT**: Протокол передачи данных.
* **Docker / Docker Compose**: Контейнеризация и оркестрация.

## 📝 Отчет по реализации
| Компонент | Детали реализации |
| :--- | :--- |
| **Инфраструктура** | Развернут локальный сервер ThingsBoard через `docker-compose`. |
| **Логика работы** | Эмулятор имитирует работу оборудования с учетом физических шумов. |
| **Обработка событий** | Настроен `Rule Chain` для фильтрации данных и создания алармов (>90% clog). |

| **Dashboard** | Скриншот загружен в репозиторий. |

## 🚀 Быстрый запуск

1. **Запуск сервера:**
   ```bash
   docker-compose up -d

После запуска откройте браузер: `http://localhost:9090` (логин/пароль: `sysadmin@thingsboard.org` / `sysadmin`).

2. **Настройка устройства**
1. В интерфейсе ThingsBoard перейдите в **Devices** -> **Add new device**.
2. Укажите имя (например, `Ventilation_System`).
3. После создания нажмите на устройство, перейдите во вкладку **Credentials** и скопируйте **Access Token**.

 3. **Запуск эмуляторa**
1. Откройте файл `emulator.py`.
2. В строке `TOKEN = "ВАШ_ТОКЕН"` вставьте скопированный токен устройства.
3. Установите зависимости и запустите скрипт:
   ```bash
   pip install paho-mqtt
   python3 emulator.py```
